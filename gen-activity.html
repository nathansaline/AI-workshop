IPv6 Compression

A step-by-step tutorial for beginners

    Step 1 of 7

    1. Welcome and Goals

    IPv6 is the newest version of the Internet Protocol. It gives every device on the internet a unique address. IPv6 addresses are long, so people use short forms to write them. This tutorial teaches you how to read and write those short forms.

    By the end of this tutorial, you will be able to:
    • Remember the two rules used to shorten (compress) an IPv6 address.
    • Understand why each rule works.
    • Apply both rules to compress a full IPv6 address correctly.

    Quick definition "Compression" here means writing the same address with fewer characters. It does not change the address. It only changes how it looks.

    2. The Structure of an IPv6 Address

    An IPv6 address has 128 bits. To make it easier to read, we split it into 8 groups. Each group has 4 hexadecimal digits (numbers 0–9 and letters a–f). Groups are separated by colons ( : ).

    2001:0db8:0000:0042:0000:8a2e:0370:7334
    Parts of an IPv6 address
    PartWhat it means
    8 groupsEach address has exactly 8 groups of digits.
    4 digits per groupEach group can have up to 4 hex digits (0–9, a–f).
    ColonsColons separate the 8 groups from each other.

    Common mix-up Do not confuse IPv6 colons ( : ) with IPv4 dots ( . ). IPv4 looks like 192.168.1.1. IPv6 looks like the example above.

    3. Rule 1 — Drop Leading Zeros

    Inside each group, you can drop (remove) zeros at the start of the group. You keep at least one digit per group.

    Example

    0042 → becomes 42
    • Look at the group: 0042
    • Find the zeros at the very start: 00
    • Remove only the leading zeros. Do not remove zeros in the middle or end.
    • Result: 42

    Common mix-up The group 0db8 becomes db8, not db. Only the leading zero is dropped. The zero is not in the middle here, so it is safe to drop just that first one.

    4. Rule 2 — Replace Zero Groups with a Double Colon

    If you have one or more groups in a row that are all 0, you can replace them with a double colon ::. This can only be done one time in an address.

    Why only once? If you used :: twice, the computer would not know how many zero groups belong to each ::. The address would be unclear. So the rule keeps the address readable.

    Example

    2001:db8:0:0:0:8a2e:370:7334 → 2001:db8::8a2e:370:7334
    • Scan the address for a run of consecutive all-zero groups.
    • Find the longest run. Here, three groups in a row are all zero.
    • Replace that whole run with ::
    • Keep every other group the same.

    Tip If there are two separate runs of zero groups, only compress the longer one with ::. Write the shorter run out normally.

    5. Full Worked Example

    Let's compress this full address using both rules, one step at a time.

    2001:0db8:0000:0000:0000:0000:1428:57ab
    • Apply Rule 1 to every group. Drop leading zeros in each group.
      2001 : db8 : 0 : 0 : 0 : 0 : 1428 : 57ab
    • Find runs of all-zero groups. Groups 3, 4, 5, and 6 are all 0. That is a run of 4 groups.
    • Apply Rule 2. Replace that run with ::
      2001:db8::1428:57ab
    • Check the result. The compressed address still represents the same 128-bit address, just written shorter.
    Final answer: 2001:db8::1428:57ab

    Why this matters Shorter addresses are easier for people to read, type, and troubleshoot. Computers still understand the full 128-bit value either way.

    7. Summary

    Great work! Here is what you learned:

    • An IPv6 address has 8 groups of hex digits, separated by colons.
    • Rule 1: Drop leading zeros inside a group.
    • Rule 2: Replace the longest run of all-zero groups with ::, and only do this once.
    • Compression changes how the address looks, not what it means.
    Keep practicing Try compressing addresses you find in networking documentation or textbooks. The more you practice, the faster you will recognize the patterns.
